# LinkTracker

Telegram-бот для отслеживания изменений на веб-страницах. Пользователь добавляет ссылки на GitHub-репозитории и вопросы StackOverflow — бот следит за обновлениями и присылает уведомления в реальном времени.

## Архитектура

Проект построен по принципу **многосервисного приложения** с разделением ответственности между независимыми сервисами.

```
+---------------+          OpenAPI          +----------------+
|      Bot      | <-----------------------> |    Scrapper    |
|    (TG UI)    |                           |  (мониторинг) |
+---------------+                           +-------+--------+
                                                    |
                                                    v
                                           +--------+-------+
                                           |     Agent      |
                                           | (обр. данных)  |
                                           +----------------+
```

Контракт взаимодействия между Bot и Scrapper описан в OpenAPI-спецификации, что позволяет менять реализацию каждого сервиса независимо.

Внутри каждого сервиса применяется **Clean Architecture** — три чётко разграниченных слоя:

- `domain` — модели предметной области, без зависимостей
- `application` — юзкейсы, бизнес-логика
- `infrastructure` — взаимодействие с внешним миром (Telegram API, HTTP, диск)

## Сервисы

**`cmd/bot`** — сервис Telegram-бота. Принимает команды от пользователя, управляет списком отслеживаемых ссылок и доставляет уведомления через [Telegram Bot API](https://core.telegram.org/bots/api).

**`cmd/scrapper`** — сервис мониторинга контента. Периодически проверяет добавленные ссылки на наличие изменений и отправляет обновления в сервис Bot.

**`cmd/agent`** — сервис интеллектуальной обработки данных.

**`cmd/report`** — служебный модуль для агрегирования отчётов сборки.

## Поддерживаемые платформы

| Платформа | Что отслеживается |
|-----------|-------------------|
| **GitHub** | Новые коммиты, issues, pull requests |
| **StackOverflow** | Новые ответы на вопрос |

## Структура проекта

```
LinkTracker/
├── cmd/
│   ├── bot/           # Точка входа сервиса Telegram-бота
│   ├── scrapper/      # Точка входа сервиса мониторинга
│   ├── agent/         # Точка входа сервиса обработки данных
│   └── report/        # Служебный модуль отчётов
├── internal/
│   ├── application/   # Юзкейсы (бизнес-логика)
│   ├── domain/        # Модели предметной области
│   └── infrastructure/# Инфраструктурные адаптеры
├── pkg/               # Переиспользуемые пакеты (внешний импорт разрешён)
├── bin/               # Скомпилированные бинарники (генерируется)
├── Makefile
├── go.mod
└── .env               # Секреты (не коммитится)
```

Разграничение `internal` и `pkg` — осознанное архитектурное решение. Пакеты из `internal` недоступны для импорта снаружи проекта. Пакеты из `pkg` независимы от остальной кодовой базы и могут переиспользоваться в других проектах.

## Конфигурация

Создайте файл `.env` в корне проекта:

```env
APP_TELEGRAM_TOKEN=123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11
```

Файл `.env` добавлен в `.gitignore` — токены никогда не попадают в репозиторий.

## Сборка и запуск

**Требования:** Go 1.25+, утилита `make`.

```bash
git clone https://github.com/Br1ght-SuN-lab/LinkTracker.git
cd LinkTracker
```

Сборка всех сервисов сразу:
```bash
make build
```

Сборка отдельного сервиса:
```bash
make build_bot
make build_scrapper
make build_agent
```

Скомпилированные бинарники появятся в директории `bin/`:
```bash
./bin/bot
./bin/scrapper
./bin/agent
```

Список доступных команд:
```bash
make help
```

## Запуск

Запуск телеграмм-бота:
```bash
make run-bot
```

Запуск сервиса scrapper:
```bash
make run-scrapper
```

P.s важно запускать в разных терминалах, пока нет docker file 

## Тесты

```bash
make test
```

Запуск включает детектор гонок (`--race`), замер покрытия по всем пакетам и вывод итогового процента покрытия.

## Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Начало работы, приветственное сообщение |
| `/help` | Список доступных команд |
| `/track <url>` | Начать отслеживать ссылку |
| `/untrack <url>` | Прекратить отслеживать ссылку |
| `/list` | Показать все отслеживаемые ссылки |

## Внешние зависимости

- [Telegram Bot API](https://core.telegram.org/bots/api) — взаимодействие с пользователем
- [GitHub REST API](https://docs.github.com/en/rest) — мониторинг репозиториев
- [StackOverflow API](https://api.stackexchange.com/docs) — мониторинг вопросов
